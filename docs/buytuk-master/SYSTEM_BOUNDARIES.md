## ✅ ما يفعله BuyTuk

### جمع الأدلة (Evidence Collection)
```
✅ تسجيل جلسات القراءة بمعايير موضوعية
✅ تسجيل جلسات الإملاء والكتابة والنطق
✅ تسجيل جلسات الطلاقة والتفكير وسلوك التعلم
✅ ربط الأدلة بمعايير تعليمية محددة
✅ حفظ تاريخ تعلم الطالب عبر الزمن
```

### قياس الإتقان (Mastery Measurement)
```
✅ حساب درجة الإتقان لكل معيار تلقائياً
✅ تصنيف مستوى الأداء (ممتاز/جيد/متوسط/ضعيف)
✅ تتبع التغير في الإتقان عبر الزمن
✅ مقارنة الأداء بمعايير الصف الدراسي
```

### التشخيص (Diagnostics)
```
✅ كشف الفجوات التعليمية تلقائياً بناءً على الأدلة
✅ تحديد شدة الفجوة (حرجة/متوسطة/بسيطة)
✅ تحليل أسباب الضعف
✅ تنبيه المعلم بالفجوات الجديدة
```

### العلاج (Remediation)
```
✅ اقتراح بروتوكول علاج مناسب لكل فجوة
✅ تتبع تقدم الطالب في خطة العلاج
✅ قياس أثر العلاج بمقارنة القبل والبعد
✅ تحديث حالة الفجوة بعد العلاج
```

### التقارير (Reporting)
```
✅ تقارير الطالب الفردية (للمعلم وولي الأمر)
✅ تقارير الصف (للمعلم)
✅ تقارير المدرسة (للمدير)
✅ تقارير الجودة المؤسسية
✅ تقارير مقارنة بمرور الوقت
```

### AI المساعد
```
✅ تحليل الإجابات وإعطاء ملاحظات تشخيصية
✅ اقتراح علاجات مخصصة
✅ تلخيص تقدم الطالب بلغة طبيعية
✅ كشف أنماط التعلم غير الظاهرة
```

## ❌ ما لا يفعله BuyTuk

### ليس نظام إدارة مدرسي (SIS)
```
❌ لا يُدير جداول الحضور والغياب اليومية
❌ لا يُدير الجداول الدراسية والمواعيد
❌ لا يُدير رسوم الطلاب والمصروفات
❌ لا يُدير الرواتب والشؤون المالية للموظفين
❌ لا يُدير عمليات القبول والتسجيل المؤسسي
```

### ليس نظام إدارة التعلم (LMS)
```
❌ لا يُقدم محتوى تعليمياً (دروس، فيديوهات، كتب)
❌ لا يُدير واجبات وتسليمات المناهج
❌ لا يُتيح للمعلم نشر مواد تدريسية
❌ لا يستبدل بيئة الفصل الدراسي الرقمية
```

### ليس شبكة اجتماعية
```
❌ لا توجد منشورات أو متابعات بين المستخدمين
❌ لا يوجد تقييم اجتماعي للمعلمين من الطلاب
❌ لا يوجد نظام "likes" أو مكافآت اجتماعية
```

### ليس نظام محاسبة أو ERP
```
❌ لا فوترة للمدارس من داخل المنصة (B2B billing خارج النطاق)
❌ لا تقارير مالية للمؤسسة
❌ لا ربط بأنظمة المحاسبة
```

### ليس نظام اختبارات عشوائية
```
❌ لا يُنشئ اختبارات عشوائية بدون ربطها بمعايير
❌ لا يُعطي درجات بدون أدلة داعمة
❌ لا يُولّد شهادات بدون تحقق المعايير
```

## 🔘 منطقة رمادية — قرارات مستقبلية

هذه الميزات ليست داخل الحدود الحالية، لكن قد تُضاف بـ ADR رسمي:

```
🔘 Gamification (نقاط، شارات، لوحات متصدرين)
   → السؤال: هل تُحسّن النتائج أم تُشتّت عنها؟
   → الشرط: دراسة أثر قبل الإضافة

🔘 Parent Chat (تواصل مباشر بين معلم وولي أمر)
   → السؤال: أم نُحيل لواتساب وأمثاله؟
   → الشرط: طلب واضح ومتكرر من المستخدمين

🔘 Content Library (مكتبة محتوى للعلاج)
   → السؤال: هل ننشئ أم نتكامل مع مصادر خارجية؟
   → الشرط: بعد توثيق حاجة حقيقية في بيانات الاستخدام

🔘 Video Recording للطلاب
   → السؤال: Privacy implications ضخمة جداً
   → الشرط: Legal review + consent framework أولاً

🔘 AI-Generated Practice Exercises
   → ضمن النطاق المستقبلي إذا ثبت أن الممارسة المولّدة تُحسن الإتقان
```
