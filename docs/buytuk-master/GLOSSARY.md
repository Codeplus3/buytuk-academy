## A

### Academic Standard (معيار أكاديمي)
```
التعريف : توصيف رسمي لما يجب أن يعرفه الطالب أو يستطيع فعله في مرحلة دراسية معينة.
المثال  : "يستطيع الطالب قراءة نص من 200 كلمة بطلاقة عند الصف الثالث."
لا تخلطه مع: Performance Category (هذه نتيجة القياس، ليست المعيار نفسه)
```

### Assessment (تقييم)
```
التعريف : عملية جمع أدلة على أداء الطالب وقياسها بمعيار محدد.
أنواعه  : Formative (تكويني — أثناء التعلم) | Summative (ختامي — بعد التعلم)
في BuyTuk: كل جلسة في أحد المحركات الستة هي Assessment.
لا تخلطه مع: Examination (اختبار رسمي مُقنَّن خارجياً)
```

### Audit Log (سجل التدقيق)
```
التعريف : سجل غير قابل للتعديل يُسجَّل فيه كل عملية حساسة في النظام.
يحتوي  : من فعل ماذا + متى + القيمة قبل + القيمة بعد
الغرض  : المساءلة والشفافية وإمكانية المراجعة
```

---

## B

### Benchmark (معيار مرجعي)
```
التعريف : نقطة مرجعية تُقاس عليها النتائج لتحديد مدى التقدم أو التأخر.
المثال  : "متوسط أداء طلاب الصف الثالث على مستوى المنطقة في معيار القراءة = 72."
في BuyTuk: يُستخدم لمقارنة أداء مدرسة بمتوسط الشبكة.
لا تخلطه مع: Target (هدف مُحدَّد يُريد الطالب الوصول إليه)
```

---

## C

### Calculator (حاسبة)
```
التعريف : وحدة برمجية بحتة تأخذ inputs وتُعيد output رقمي — بدون تخزين ولا React.
القاعدة : Calculator لا يعرف قاعدة البيانات ولا الواجهة.
المثال  : masteryCalculator.ts — يأخذ درجات ويُعيد overallScore + performanceCategory
```

### Cohort (مجموعة)
```
التعريف : مجموعة طلاب يشتركون في خاصية (صف، سنة التخرج، معلم).
الاستخدام: لمقارنة الأداء بين مجموعات متجانسة.
```

### Criterion (معيار فرعي، ج: Criteria)
```
التعريف : بُعد واحد محدد يُقاس ضمن محرك الذكاء.
المثال  : في محرك القراءة — comprehension و fluency و accuracy هي Criteria.
لا تخلطه مع: Standard (المعيار الأكاديمي الأشمل)
```

---

## D

### Diagnosis / Diagnostics (تشخيص)
```
التعريف : تحليل منظَّم لأدلة أداء الطالب لتحديد جذور الضعف.
في BuyTuk: Diagnostics Engine يفحص MasteryRecords ويُنشئ LearningGaps.
القاعدة : لا تشخيص بدون أدلة — الرأي الشخصي ليس تشخيصاً.
```

### Domain Model (نموذج المجال)
```
التعريف : خريطة الكيانات الأساسية في النظام وعلاقاتها ببعضها.
المرجع  : DOMAIN_MODEL.md
```

---

## E

### Evidence (دليل)
```
التعريف : سجل موضوعي ومُحدَّد يُثبت مستوى أداء الطالب في معيار معين.
الخصائص: - مرتبط بطالب محدد + معيار محدد + وقت محدد
          - غير قابل للتعديل بعد التسجيل (immutable)
          - قابل للإحالة والمراجعة
المثال  : جلسة قراءة سُجِّلت في 2026-07-18 بدرجة 72 في معيار ENG-R-001.
لا تخلطه مع: Score (الدرجة المجردة بلا سياق)
```

### Evidence Bundle (حزمة أدلة)
```
التعريف : مجموعة من Evidence Records تخص طالباً ومعياراً في فترة زمنية.
الاستخدام: تُستخدَم معاً لحساب MasteryRecord.
```

## G

### Gap (فجوة) — اختصار لـ Learning Gap
```
انظر: Learning Gap
```

### GES Protocol (بروتوكول GES)
```
التعريف : Global Education Standard — المعيار المفتوح المقترح لقياس جودة التعلم.
الحالة  : قيد التطوير — يُتوقع إطلاقه في Phase G-1 (السنة 3-4)
الهدف   : أي منصة تعليمية في العالم تستطيع التوافق مع هذا المعيار.
```

## I

### Impact (أثر)
```
التعريف : التغيُّر القابل للقياس في مستوى إتقان الطالب نتيجة تدخل تعليمي.
الحساب  : Impact = After Score − Before Score
التصنيف : Significant (≥20) | Moderate (10-19) | Minimal (<10) | Negative (سالب)
لا تخلطه مع: Progress (تقدم عام قد لا يُنسَب لتدخل محدد)
```

### Indicator (مؤشر)
```
التعريف : سلوك قابل للملاحظة يُثبت تحقيق المعيار.
المثال  : "يقرأ الطالب 100 كلمة في الدقيقة بدون أخطاء" = مؤشر لمعيار الطلاقة.
العلاقة : Standard يحتوي على Indicators.
```

### Institution (مؤسسة)
```
التعريف : أي كيان تعليمي يستخدم BuyTuk — مدرسة، معهد، جامعة.
في الكود: يُمثَّل كـ School + Tenant.
```

### Intelligence Engine (محرك الذكاء)
```
التعريف : نظام متخصص لجمع وتحليل أدلة تعلُّم محددة بمنهجية موحَّدة.
المحركات: Reading | Dictation | Writing | Pronunciation | Fluency & Thinking | Learning Behavior
البنية  : Calculator → Service → Routes → Panels (طبقات منعزلة)
لا تخلطه مع: AI (المحركات لا تعتمد على AI — AI مكوّن إضافي)
```

---

## L

### Learning Behavior (سلوك التعلم)
```
التعريف : أنماط تفاعل الطالب مع عملية التعلم — مثل المشاركة والتركيز والاستمرارية.
محرك    : Learning Behavior Engine يقيس: participation + focus + consistency
```

### Learning Gap (فجوة تعلمية)
```
التعريف : ضعف موثَّق بالأدلة في معيار محدد، يستوجب تدخلاً علاجياً.
الشرط   : جلستا تقييم متتاليتان بدرجة < 60 في نفس المعيار.
الشدة   : Critical (< 30) | Moderate (30-44) | Minor (45-59)
دورة حياتها: active → in_remediation → resolved
لا تخلطه مع: Weakness (ضعف غير موثَّق، مجرد ملاحظة معلم)
```

---

## M

### Mastery (إتقان)
```
التعريف : مستوى تمكُّن الطالب من معيار معين، محسوب من أدلة موضوعية.
الحساب  : MasteryRecord ← يُحسَب من Evidence Bundle
لا تخلطه مع: Grade (درجة مدرسية قد تكون ذاتية)
```

### MasteryRecord (سجل الإتقان)
```
التعريف : كيان في قاعدة البيانات يحفظ درجة إتقان طالب لمعيار في وقت محدد.
الحقول  : student_id + standard_id + overall_score + performance_category + computed_at
```

### Multi-Tenancy (تعدد المستأجرين)
```
التعريف : قدرة النظام على خدمة مؤسسات متعددة مع عزل بياناتها بشكل كامل.
في BuyTuk: كل مدرسة = Tenant منفصل. مدرسة A لا ترى بيانات مدرسة B أبداً.
```

## P

### Performance Category (فئة الأداء)
```
التعريف : تصنيف نوعي لدرجة الإتقان.
القيم   : Excellent (90-100) | Good (75-89) | Average (60-74) | Weak (45-59) | Critical (<45)
المرجع  : DECISION_LOGIC.md — قسم "منطق تصنيف الأداء"
```

### Pilot (تجربة مبدئية)
```
التعريف : أول تشغيل حقيقي للمنصة في مؤسسة واحدة لجمع بيانات الاستخدام الواقعي.
الهدف   : إثبات القيمة وكشف المشاكل قبل التوسع.
المرجع  : MASTER_ROADMAP.md → Phase B-0
```

---

## R

### Remediation (علاج)
```
التعريف : تدخل تعليمي منظَّم يهدف لإغلاق فجوة تعلمية محددة.
المكونات: RemediationProtocol (بروتوكول) + RemediationAssignment (مهمة مُسنَدة)
القاعدة : لا علاج بدون تشخيص — كل Remediation مرتبطة بـ LearningGap.
```

### RemediationAssignment (مهمة علاجية)
```
التعريف : تكليف رسمي يُسنَد للطالب بخطة علاج محددة لفجوة معينة.
الحالة  : assigned → in_progress → completed | cancelled
```

### RemediationProtocol (بروتوكول علاج)
```
التعريف : مجموعة أنشطة وإجراءات مُوصى بها لعلاج نوع معين من الفجوات.
طبيعته  : يُقترَح تلقائياً — يُطبَّق بعد موافقة المعلم.
```

## S

### Service (خدمة)
```
التعريف : وحدة برمجية تُنسِّق بين Calculator وقاعدة البيانات.
القاعدة : Service لا يحتوي منطق حساب — يستدعي Calculator ويُخزِّن النتيجة.
المثال  : masteryService.ts يستدعي masteryCalculator ثم يُخزِّن MasteryRecord.
```

### Standard (معيار)
```
انظر: Academic Standard
```

### Sprint (سبرنت)
```
التعريف : وحدة عمل زمنية منظَّمة تنتهي بـ Deliverable قابل للاختبار.
في BuyTuk: كل Sprint يُضيف محركاً أو طبقة جديدة — لا يُعدِّل السابق.
```

---

## T

### Tenant (مستأجر)
```
التعريف : مؤسسة تعليمية مستقلة تستخدم BuyTuk ضمن بيئة معزولة خاصة بها.
تقنياً  : كل صف (row) في DB يحمل tenant_id الذي ينتمي إليه.
```

---

## V

### Validity (صدق)
```
التعريف : مدى قياس الأداة لما صُمِّمت لقياسه فعلاً.
المثال  : هل محرك القراءة يقيس فهم القراءة فعلاً، أم يقيس شيئاً آخر؟
الأهمية : ضروري للتحقق الأكاديمي في Phase G-0
```

### Reliability (موثوقية)
```
التعريف : مدى ثبات نتائج الأداة عند تطبيقها في ظروف متماثلة.
المقياس : Cronbach's Alpha ≥ 0.8 (الهدف في Phase G-0)
الأهمية : بدون موثوقية — المعيار لا يُصدَّق علمياً
```
