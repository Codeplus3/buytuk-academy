## رحلة المعلم — التقييم اليومي

```
الخطوة 1: تسجيل الدخول
  المعلم يفتح المنصة → يُدخل البريد وكلمة المرور
  النظام → يتحقق من الـ Credentials → يُعيد JWT Token
  ← المعلم يُوجَّه لـ Teacher Portal

الخطوة 2: اختيار الصف والطالب
  المعلم → يرى قائمة صفوفه
  يختار صف → يرى قائمة طلابه
  يختار طالب → يرى ملف الطالب (آخر نتائج، فجوات، علاجات)

الخطوة 3: تسجيل جلسة تقييم
  يختار المحرك (Reading, Dictation, Writing...)
  يُدخل درجات المعايير
  [اختياري] يُدخل ملاحظات نصية
  [اختياري] يطلب تحليل AI
  يضغط "حفظ"

الخطوة 4: عرض النتيجة
  النظام يحسب overallScore + performanceCategory
  يعرض نتيجة مباشرة
  إذا وُجدت فجوة جديدة → تنبيه للمعلم

الخطوة 5: مراجعة الفجوات
  المعلم يرى قائمة الفجوات النشطة
  يختار فجوة → يرى بروتوكول العلاج المقترح
  يوافق أو يعدّل → ينشئ RemediationAssignment

الوقت المستهدف لإتمام الخطوات 1-4: < 5 دقائق
Success Metric: المعلم يُكمل جلسته بدون رجوع لأي شخص
```

---

## رحلة المعلم — متابعة خطط العلاج

```
الخطوة 1: لوحة الفجوات النشطة
  المعلم → يرى جميع الفجوات النشطة في صفه
  مُرتَّبة حسب الشدة (Critical أولاً)
  مع تاريخ انشاء كل فجوة

الخطوة 2: تحديث تقدم الطالب
  يختار طالباً → يرى خطة علاجه
  يُحدّث نسبة الإنجاز
  يُضيف ملاحظات

الخطوة 3: إغلاق الفجوة
  بعد انتهاء العلاج → تقييم جديد
  إذا تحسّنت النتيجة → يُغلق الفجوة
  النظام يحسب ImpactMeasurement تلقائياً
  المعلم يرى: "تحسُّن 28 نقطة — أثر كبير ✓"
```

## رحلة الطالب — عرض تقدمه

```
الخطوة 1: تسجيل الدخول
  الطالب → يُدخل بياناته → يُوجَّه لـ Student Portal

الخطوة 2: لوحة التقدم الشخصي
  يرى: مستوى إتقانه لكل معيار (بصري وواضح)
  يرى: هل هناك خطة علاج جارية؟
  يرى: تقدمه منذ الشهر الماضي

الخطوة 3: تفاصيل الأداء
  يختار معيار → يرى تاريخ درجاته
  يرى: هل تحسّن أم تراجع؟
  يرى: ماذا ينصحه النظام بالتركيز عليه؟

النمط المطلوب:
✅ الطالب يفهم مستواه بدون شرح من المعلم
✅ الطالب يرى تقدمه بشكل محفِّز
✗ لا أرقام مجردة بدون سياق
```

## رحلة ولي الأمر — متابعة ابنه

```
الخطوة 1: تسجيل الدخول
  ولي الأمر → يُوجَّه لـ Parent Portal

الخطوة 2: نظرة عامة
  يرى: هل ابنه يسير بشكل طبيعي؟
  بالأخضر/الأصفر/الأحمر — لا أرقام معقدة
  يرى: هل هناك فجوات حرجة تحتاج اهتمامه؟

الخطوة 3: تفاصيل (عند الحاجة)
  يضغط "عرض التفاصيل" → يرى المعايير والدرجات
  يرى: "ابنك أحرز تحسناً في القراءة بنسبة 18% هذا الشهر"

النمط المطلوب:
✅ لغة بشرية، ليست تقنية
✅ تركيز على: ماذا يحتاج الطالب من البيت؟
✗ لا إرهاق بالأرقام والمعادلات
```

## رحلة مدير المدرسة — الصورة الكاملة

```
الخطوة 1: لوحة القرار
  المدير → يرى صورة المدرسة دفعة واحدة:
  - نسبة الطلاب في كل مستوى أداء
  - أكثر المعايير ضعفاً في المدرسة
  - فجوات حرجة غير معالجة
  - تحسن الجودة هذا الفصل vs الفصل الماضي

الخطوة 2: التعمق في الصف الأسوأ
  يختار صف → يرى معلمه + أداء طلابه
  يرى: هل المعلم يسجّل جلسات بانتظام؟
  يرى: هل الفجوات تُعالَج؟

الخطوة 3: استخراج تقرير
  تقرير الجودة المؤسسي → يُصدَّر كـ PDF
  جاهز للوزارة أو اجتماع مجلس الأمناء

النمط المطلوب:
✅ القرار في ثوانٍ، لا دقائق
✅ تقرير جاهز للطباعة في نقرة واحدة
✗ لا تنقل بين 5 شاشات لرؤية صورة المدرسة
```

## رحلة الإعداد الأولي للمدرسة

```
الخطوة 1: تسجيل المدرسة
  Admin/BuyTuk Team → ينشئ Tenant جديد
  يُدخل: اسم المدرسة، المنطقة، عدد الطلاب المتوقع

الخطوة 2: إنشاء حساب المدير
  → يُرسَل بريد ترحيبي + رابط تفعيل

الخطوة 3: إعداد الصفوف والمعلمين
  المدير → يُنشئ الصفوف
  يُضيف المعلمين (يدوياً أو بـ CSV Import)

الخطوة 4: إضافة الطلاب
  المعلمون → يُضيفون طلابهم (أو CSV Import)

الخطوة 5: أول جلسة تقييم
  → المدرسة جاهزة

الوقت المستهدف من البداية لأول جلسة: < 30 دقيقة
Success Metric: مدير يُعدّ المدرسة وحده بدون تدخل فريق BuyTuk
```
