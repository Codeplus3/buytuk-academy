## 1. منطق حساب الدرجة الإجمالية

### القاعدة العامة لجميع المحركات
```
overallScore = average(criterion_1, criterion_2, ..., criterion_n)

السبب: كل معيار يقيس بُعداً مستقلاً — لا يُطغى بُعد على آخر.
       المتوسط البسيط يضمن العدالة والقابلية للتفسير.

الاستثناء: إذا ثبت بحثياً أن بعض المعايير أهم من غيرها،
           يُضاف وزن (weight) موثَّق بـ ADR.
```

### درجات المحركات الستة

```
Reading Engine:
overallScore = avg(comprehension, fluency, accuracy)
النطاق: 0-100 لكل معيار

النطاق: 0-100 لكل معيار

Writing Engine:

النطاق: 0-100 لكل معيار
السبب: الكتابة تحتاج 4 أبعاد — أقل يُضيّع فروقاً جوهرية

النطاق: 0-100 لكل معيار
```

## 2. منطق تصنيف الأداء (Performance Category)

### العتبات الافتراضية
```
90-100 : ممتاز (Excellent)
75-89  : جيد جداً (Good)
60-74  : متوسط (Average)
45-59  : ضعيف (Weak)
0-44   : ضعيف جداً (Critical)
```

### لماذا هذه العتبات؟
```
المبدأ: العتبات مُعيَّرة على توزيع أداء الطلاب في السياق التعليمي العربي.
        تبدأ كقيم افتراضية معقولة، وتُعدَّل بعد جمع بيانات حقيقية.

التعديل المستقبلي:
- بعد 1000+ جلسة: تُحسَب percentiles حقيقية
- يُطلب ADR قبل تغيير أي عتبة
- التغيير لا يُطبَّق بأثر رجعي (backward compatible)

ما لا يُفعل:
✗ العتبات لا تتغير لطلب واحد بدون ADR
✗ لا عتبة مختلفة لكل معلم (تفضيلات شخصية تُفسد قابلية المقارنة)
```

## 3. منطق التشخيص (Diagnostics)

### متى تُفتح فجوة تعلمية؟
```
الشرط: overallScore < 60 في جلستين متتاليتين للمعيار نفسه

السبب:
- جلسة واحدة ضعيفة قد تكون ظرفاً (تعب، غياب، قلق)
- جلستان متتاليتان = نمط متكرر = دليل كافٍ
- المعيار: "جلستان" وليس "جلسة واحدة" يُقلل من الإنذارات الكاذبة

استثناء (فجوة حرجة):
إذا كان overallScore < 30 في أي جلسة → تُفتح فجوة حرجة فوراً
السبب: ضعف حاد يحتاج تدخلاً عاجلاً، لا ينتظر التأكيد
```

### كيف تُحدَّد شدة الفجوة؟
```
critical (حرج)  : overallScore < 30
moderate (متوسط): overallScore 30-44 في جلستين
minor (بسيط)    : overallScore 45-59 في جلستين

السبب: الشدة تُحدد أولوية التدخل وطبيعة العلاج.
       "حرج" يعني المعلم يتصرف اليوم.
       "بسيط" يعني تُضاف لقائمة الاهتمام.
```

## 4. منطق اختيار العلاج (Remediation)

### كيف يختار النظام العلاج؟
```
1. يُحدَّد standard_id + gap_severity
2. يُبحث في RemediationProtocols عن بروتوكول مطابق:
   - أولاً: مطابقة standard_id + severity
   - ثانياً: مطابقة subject + severity (fallback)
   - ثالثاً: بروتوكول عام للـ severity فقط (آخر خيار)

3. يُعرض على المعلم — ليس يُطبَّق تلقائياً
4. المعلم يوافق أو يعدّل أو يرفض

السبب:
AI + Rules يقترحان — الإنسان يقرر.
المعلم يعرف سياق الطالب أكثر من أي خوارزمية.
```

### قاعدة العلاج الواحد
```
طالب واحد + معيار واحد = خطة علاج واحدة نشطة في وقت واحد

السبب: تعدد الخطط يُربك المعلم ويُرهق الطالب.
       إغلاق الخطة الحالية قبل فتح جديدة.
       الاستثناء: معايير مختلفة تماماً يمكن علاجها بالتوازي.
```

## 5. منطق قياس الأثر (Impact Measurement)

### متى يُقاس الأثر؟
```
الشرط: إغلاق RemediationAssignment بحالة "completed"

الحساب:
before_score = overallScore من أول جلسة أدت للفجوة
after_score  = overallScore من جلسة بعد انتهاء العلاج
delta        = after_score - before_score
```

### كيف تُحدَّد أهمية الأثر؟
```
significant  : delta ≥ 20 نقطة
moderate     : delta 10-19 نقطة
minimal      : delta < 10 نقطة
negative     : delta سالب (تراجع)

السبب: 20 نقطة تعني انتقال حقيقي من فئة أداء لأخرى.
       10 نقطة تحسن ملحوظ لكن غير كافٍ للانتقال.
       أقل من 10 قد يكون ضمن هامش التقلب الطبيعي.
```

### لماذا لا نُلغي الفجوة تلقائياً؟
```
القرار: إغلاق الفجوة يتطلب موافقة المعلم — ليس تلقائياً.

السبب:
- overallScore 65 بعد العلاج = أعلى من عتبة الفجوة
- لكن المعلم قد يرى أن الطالب يحتاج مزيداً من التعزيز
- القرار البشري يبقى السلطة الأخيرة
```

## 6. منطق AI في المنصة

### ماذا يفعل AI؟
```
يفعل:
✅ يقترح تشخيصاً بناءً على نمط الأدلة
✅ يقترح بروتوكول علاج مناسب
✅ يُلخص تقدم الطالب بلغة طبيعية
✅ يُشير لأنماط يصعب ملاحظتها يدوياً

لا يفعل:
✗ لا يُعطي درجات مباشرة للطالب
✗ لا يُلغي أو يُنشئ فجوات بدون مراجعة بشرية
✗ لا يُخبر ولي الأمر مباشرة بنتيجة سلبية
✗ لا يتخذ قرار ترفيع أو رسوب
```

### حدود ثقة AI
```
AI Confidence Score يُعرض دائماً:

HIGH (≥ 80%): "النظام واثق من هذا الاقتراح"
MEDIUM (60-79%): "اقتراح معقول — راجع البيانات"
LOW (< 60%): "بيانات غير كافية — حكم المعلم أولى"

السبب:
النظام الذي يُخفي عدم اليقين أخطر من النظام الذي يُظهره.
```

## 7. قواعد لا تُخترق

```
R-01: لا قرار على طالب بدون دليل مُسجَّل في قاعدة البيانات
R-02: لا يُعدَّل سجل Evidence بعد إنشائه (immutable)
R-03: لا تُستخدم بيانات طالب لاتخاذ قرار في شأن طالب آخر
R-04: كل قرار يُسجَّل في Audit Log مع: من + ماذا + متى + لماذا
R-05: الخطأ يُسجَّل بشفافية — لا يُخفى ولا يُصحَّح بصمت
R-06: لا تغيير في خوارزميات الحساب بدون ADR + إعلام المستخدمين
```
