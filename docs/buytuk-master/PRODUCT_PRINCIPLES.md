```
الإصدار : 1.0.0
التاريخ  : 2026-07-18

السلطة   : تعديل هذه المبادئ يتطلب ADR + تصويت فريق المنتج
```

---

## PP-01: الأدلة قبل الدرجات
```
المبدأ: لا نُعطي حكماً على طالب بدون دليل موضوعي مُسجَّل.
        الدرجة بدون دليل = رأي، لا قياس.

التطبيق:
✅ كل تقييم يُنشئ Evidence Record في DB
✅ يمكن للمعلم والطالب وولي الأمر رؤية الدليل
✗ لا نُظهر درجة إجمالية بدون إظهار مصدرها

اختبار: "هل يمكن لأحد أن يسأل: من أين جاءت هذه الدرجة؟"
         إذا كانت الإجابة "لا أعرف" → المبدأ مُنتهَك.
```

---

## PP-02: التشخيص قبل العلاج
```
المبدأ: لا نقترح علاجاً بدون تشخيص واضح للسبب.
        العلاج العشوائي يُضيع وقت المعلم والطالب.

التطبيق:
✅ RemediationAssignment تُنشأ دائماً من LearningGap
✗ لا "علاج عام" بدون فجوة محددة
✗ لا يوصي النظام بـ "ممارسة أكثر" بدون تحديد ما يجب ممارسته

اختبار: "هل يعرف المعلم لماذا يُعطى هذا العلاج بالذات؟"
```

---

## PP-03: AI يساعد — الإنسان يقرر
```
المبدأ: كل قرار يؤثر على الطالب يتطلب موافقة بشرية.
        AI أداة تضخيم — ليس سلطة.

التطبيق:
✅ AI يقترح تشخيصاً → المعلم يوافق أو يرفض
✅ AI يقترح علاجاً → المعلم يعدّل أو يقبل
✗ لا قرار يُلزم الطالب يُنفَّذ تلقائياً بدون مراجعة بشرية
✗ لا إغلاق فجوة تلقائي بدون موافقة معلم

اختبار: "هل يمكن لـ AI أن يغير مسار الطالب التعليمي بدون تدخل إنسان؟"
         إذا "نعم" → المبدأ مُنتهَك.
```

---

## PP-04: البساطة تقود القيمة
```
المبدأ: الميزة المعقدة التي لا يستخدمها المعلم = صفر قيمة.
        الاستخدام يسبق الكمال.

التطبيق:
✅ كل ميزة جديدة يُقاس مدى استخدامها بعد شهر
✅ مهمة التقييم تكتمل في أقل من 5 دقائق
✗ لا نُضيف خطوة إذا يمكن تجنبها
✗ لا نُبالغ في الخيارات — الإعدادات الصحيحة افتراضياً

اختبار: "هل يستطيع معلم جديد إتمام أول تقييم بدون تدريب؟"
```

---

## PP-05: كل ميزة تحل مشكلة حقيقية
```
المبدأ: "يبدو مفيداً" ليس سبباً كافياً للبناء.
        يجب أن توجد مشكلة موثَّقة ومستخدم يعاني منها.

التطبيق:
✅ طلبات الميزات تُسجَّل في FUTURE_IDEAS.md
✅ الميزة تُبنى عندما يطلبها 3+ مستخدمين مختلفين بشكل مستقل
✗ لا نبني ميزة لأننا "نتوقع" أن تكون مفيدة
✗ لا نُقلِّد منافسين بدون مبرر حقيقي

اختبار: "من الذي يعاني من المشكلة التي تحلها هذه الميزة؟"
         "كم مرة قالها؟"
```

---

## PP-06: لا قرار على طالب بدون شفافية
```
المبدأ: الطالب وولي الأمر يحق لهم فهم سبب أي حكم على الطالب.
        المنصة لا تعمل كـ "صندوق أسود".

التطبيق:
✅ كل نتيجة تُوضَّح معاييرها (من أين جاءت؟)
✅ كل تشخيص يُوضَّح مبناه (على أي أدلة؟)
✅ كل اقتراح AI يُوضَّح مستوى ثقته
✗ لا ننقل نتائج للأهل بلغة تقنية — تُترجم للبشرية

اختبار: "هل يستطيع ولي الأمر أن يفهم سبب النتيجة بدون مساعدة؟"
```

---

## PP-07: الخصوصية بالتصميم — ليست إضافة لاحقاً
```
المبدأ: بيانات الأطفال حساسة جداً — تُعامَل كذلك من اليوم الأول.

التطبيق:
✅ لا مشاركة بيانات شخصية بدون موافقة صريحة
✅ البيانات المُجمَّعة مُجهَّلة تماماً
✅ كل مستخدم يرى فقط ما يحق له رؤيته
✗ لا بيانات شخصية في الـ Logs
✗ لا تصدير بيانات طلاب بدون تسجيل في Audit

اختبار: "هل يمكن لشخص غير مخوَّل أن يصل لبيانات طالب بأي طريقة؟"
```

---

## PP-08: الاستمرارية تحمينا من أنفسنا
```
المبدأ: المشروع يجب أن يعيش بعد الفريق الحالي.
        ما لا يُوثَّق — يُفقَد.

التطبيق:
✅ كل قرار في ADR
✅ كل محرك موثَّق في DECISION_LOGIC.md
✅ كل Sprint ينتهي بـ HANDOVER.md محدَّث
✗ لا "معرفة ضمنية" في رؤوس الأشخاص فقط

اختبار: "هل يستطيع مطور جديد أن يفهم القرار وسببه بدون سؤال أحد؟"
```

---

## PP-09: الأداء جزء من المنتج — ليس تحسيناً لاحقاً
```
المبدأ: المنصة البطيئة = منصة لا يستخدمها أحد.

التطبيق:
✅ API response: < 200ms (p95)
✅ Page load: < 3 ثوانٍ على شبكة متوسطة
✅ كل ميزة تُختبر بأداء واقعي قبل النشر
✗ "سنُسرّعها لاحقاً" — لا، نبنيها صحيحة من البداية

اختبار: "هل يشعر المعلم بالانتظار عند استخدام المنصة؟"
```

---

## PP-10: نقيس استخدامنا كما نقيس تعلم الطلاب
```
المبدأ: لا نعرف أن المنتج يعمل ما لم نقس استخدامه.

مؤشرات أساسية:
✅ عدد الجلسات المسجَّلة / معلم / أسبوع
✅ وقت إتمام أول تقييم (للمستخدمين الجدد)
✅ نسبة العلاجات المكتملة
✅ معدل العودة للمنصة (Retention)
✗ لا نبني feature بدون قياس أثرها على المؤشرات
```

## قائمة التحقق عند أي طلب ميزة جديدة

```
قبل قول "نعم" لأي ميزة جديدة — أجب على هذه الأسئلة:

[ ] هل تحل مشكلة حقيقية موثَّقة؟ (PP-05)
[ ] هل تتوافق مع SYSTEM_BOUNDARIES.md?
[ ] هل تلتزم بـ PP-03 (AI مساعد لا حاكم)؟
[ ] هل تلتزم بـ PP-07 (خصوصية البيانات)؟
[ ] هل المعلم سيستخدمها في < 5 دقائق؟ (PP-04)
[ ] هل يمكن شرح نتيجتها لولي الأمر بلغة بشرية؟ (PP-06)

إذا أُجيب بـ "لا" على أي سؤال → يعود الطلب للمراجعة.
```
