## خريطة القيمة (Value Map)

### من يعاني من المشكلة؟

```
المعلم:
  "أعرف أن طالبي ضعيف، لكن لا أعرف بالضبط أين ولماذا."
  "أقضي ساعات في تصحيح، لكن لا أعرف إذا التصحيح أفاد."

مدير المدرسة:
  "لا أعرف ما يجري في الفصول فعلاً إلا من كلام المعلمين."
  "عند التقييمات — أرى المشكلة بعد فوات الأوان."

ولي الأمر:
  "أعرف أن ابني لديه تقدير A أو B، لكن لا أعرف ما يعني ذلك فعلاً."
  "لا أعرف كيف أساعده في البيت."

وزارة التعليم (مستقبلاً):
  "لا توجد بيانات موضوعية وقابلة للمقارنة عبر المدارس."
```

### القيمة التي تُقدمها BuyTuk

```
للمعلم         : تشخيص دقيق + خطة علاج واضحة → يعرف ماذا يفعل
لمدير المدرسة  : صورة كاملة وموضوعية → يتخذ قرارات بالبيانات
لولي الأمر     : فهم حقيقي لأداء ابنه + ماذا يفعل في البيت
للطالب         : مسار تعلم شخصي + تقدم مرئي يُحفّزه
للمؤسسة التعليمية: تقارير جودة موثوقة للاعتماد والتقييم
```

## ثلاثة أدوار — ثلاثة مستويات من العلاقة

```
Buyer     (المشتري):   مدير المدرسة / إدارة التعليم
           → يتخذ قرار الشراء + يوقع العقد

Champion  (المروّج):  مدير المدرسة + معلم متحمس
           → يُروّج داخلياً + يُقنع الزملاء

User      (المستخدم): المعلم + الطالب + ولي الأمر
           → يستخدم يومياً + يُقيّم الجودة

BuyTuk يجب أن يُرضي الثلاثة بشكل مختلف:
  للـ Buyer: ROI، تقارير، compliance، سهولة الشراء
  للـ Champion: إثبات النجاح، بيانات للمدير، بطل داخلي
  للـ User: سرعة، سهولة، قيمة مرئية في دقائق
```

## الميزة التنافسية

```
ما يملكه البيتوك ما لا يملكه غيره بسهولة:

1. المحركات الستة المتكاملة
   → منافسون لديهم أداة أو اثنتين، لا ست متكاملة

2. خط Evidence → Mastery → Diagnosis → Remediation → Impact
   → خط كامل من التشخيص للأثر — لا أحد بنى هذا للعربية

3. التوثيق المؤسسي الكامل
   → يمكن دخول شريك أو مستثمر خلال 48 ساعة

4. القابلية للتوسع (Architecture)
   → Multi-tenant + API-first = جاهز للـ Enterprise

الحماية:
  Network Effect: كلما زادت المدارس، كلما صارت البيانات المجمّعة أثمن
  Switching Cost: الأدلة والتاريخ محفوظ عندنا — الانتقال يعني فقدان التاريخ
  Data Moat: بيانات حقيقية عن تعلم الطلاب العربي — لا أحد لديه هذا
```

## Unit Economics (اقتصاديات الوحدة)

```
المؤشرات الهدف (Phase T-2 — بعد Pilot):

CAC (Customer Acquisition Cost): تكلفة اكتساب مدرسة واحدة
  الهدف: < ربع قيمة ACV (Annual Contract Value)

ACV (Annual Contract Value): متوسط العقد السنوي
  الهدف: حسب Pricing_Model.md

LTV (Lifetime Value): قيمة العميل على مدى حياته
  الفرضية: 3 سنوات × ACV = LTV
  الهدف: LTV/CAC > 3

Churn Rate (معدل الخسارة):
  الهدف: < 10% سنوياً في النهاية
  في البداية: Pilot data يُحدد التوقع الحقيقي

Gross Margin:
  الهدف: > 70% (SaaS عادي)
  في البداية: < 50% (hosting + support يدوي)
```

## افتراضات يجب التحقق منها (Assumptions to Validate)

```
الافتراض 1: المدارس الخاصة تدفع مقابل بيانات دقيقة
→ كيف نتحقق: Pilot بسعر رمزي — هل يدفعون؟

الافتراض 2: المعلمون يستخدمون المنصة بانتظام (لا يتركونها)
→ كيف نتحقق: Retention بعد أسبوعين من Pilot

الافتراض 3: البيانات تُحسّن نتائج الطلاب فعلاً
→ كيف نتحقق: مقارنة Before/After في PILOT_PLAN.md

الافتراض 4: المدارس تُجدد الاشتراك بعد سنة
→ كيف نتحقق: لا يمكن قبل السنة الثانية — لكن NPS يُشير

هذه الافتراضات إذا أُثبتت → النمو ممكن
إذا نُفيت → يجب مراجعة النموذج
```
