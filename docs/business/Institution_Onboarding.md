## استقبال المؤسسة تقنياً | الخطوات الداخلية لإعداد Tenant جديد

---

```
الإصدار : 1.0.0
التاريخ  : 2026-07-18
الغرض   : دليل داخلي لفريق BuyTuk — كيف تُعدّ مدرسة جديدة تقنياً
```

---

## المتطلبات قبل البدء

```
□ عقد موقَّع (أو اتفاق Pilot مكتوب)
□ البيانات التالية من المدرسة:
  - اسم المؤسسة الرسمي
  - اسم المدير ومساعده (إن وجد)
  - البريد الإلكتروني الرسمي
  - عدد الطلاب التقريبي
  - عدد المعلمين
  - قائمة الفصول (الصف + الشعبة + المعلم المسؤول)
  - قائمة المعلمين (الاسم + البريد + الفصول)
  - قائمة الطلاب (الاسم + الرقم الدراسي + الفصل)
  - الخطة المشتراة (Starter / Growth / School / District)
```

---

## الخطوة 1: إنشاء الـ Tenant

```bash
# في Admin Portal أو مباشرة في DB (استخدم Admin Portal دائماً)

البيانات المطلوبة:
  name      : "مدرسة الرواد الخاصة"
  slug      : "alruwad-private"  (يُستخدم في URL — بدون مسافات)
  plan_id   : "starter" | "growth" | "school" | "district"
  is_active : true
  region    : "الرياض" أو "جدة" أو ...
  city      : "الرياض"

بعد الإنشاء:
  □ سجّل tenant_id الناتج في ملف تتبع العملاء
  □ تحقق: GET /api/health (Tenant يظهر في DB)
```

---

## الخطوة 2: إنشاء حساب المدير

```
في Admin Portal → Users → Add User:

  first_name    : [اسم المدير]
  last_name     : [اسم العائلة]
  email         : [البريد الرسمي]
  role          : "principal"
  tenant_id     : [ID المدرسة]
  is_active     : true
  temp_password : [كلمة مرور مؤقتة قوية — 12+ حرف]

بعد الإنشاء:
  □ انسخ: البريد + كلمة المرور المؤقتة
  □ أرسل Welcome Email مع هذه البيانات (Customer_Onboarding.md)
  □ سجّل في CRM: حساب المدير جاهز ✓
```

---

## الخطوة 3: إنشاء الفصول

```
لكل فصل دراسي:

  name         : "الثالث أ" أو "Grade 3 - Section A"
  grade_level  : "3" أو "Grade 3"
  section      : "أ" أو "A"
  school_id    : [ID المدرسة]
  tenant_id    : [ID المدرسة]
  academic_year: "2026-2027"

إذا عدد الفصول > 5 → استخدم CSV Import:
  grade_level, section, academic_year
  "3", "أ", "2026-2027"
  "3", "ب", "2026-2027"
  ...
```

## الخطوة 4: استيراد المعلمين

```
أسرع طريقة: CSV Import

Template (المعلمون):
  first_name, last_name, email, role, class_ids
  "سارة", "العتيبي", "sara@school.com", "teacher", "class-001"
  "محمد", "الشهري", "mohammed@school.com", "teacher", "class-002,class-003"

ملاحظات:
  □ class_ids تُفصَل بفاصلة إذا المعلم يُدرّس أكثر من فصل
  □ كلمات المرور المؤقتة تُولَّد تلقائياً → تُرسَل لكل معلم بريداً

بعد الاستيراد:
  □ تحقق: عدد المعلمين المُضافين = عدد الملفات ✓
  □ تحقق: كل معلم مرتبط بفصله الصحيح ✓
```

---

## الخطوة 5: استيراد الطلاب

```
Template (الطلاب):
  first_name, last_name, student_id, class_id
  "عمر", "الزهراني", "STU-001", "class-001"
  "نور", "المالكي", "STU-002", "class-001"
  "فيصل", "القحطاني", "STU-003", "class-002"

ملاحظات:
  □ student_id يجب أن يكون فريداً داخل الـ Tenant
  □ class_id هو ID الفصل المُنشَأ في الخطوة 3
  □ لا حسابات للطلاب في Phase الأولى (العرض عبر Teacher فقط)

بعد الاستيراد:
  □ تحقق: إجمالي الطلاب = عدد المُستوردين ✓
  □ تحقق: توزيع الطلاب على الفصول صحيح ✓
```

---

## الخطوة 6: التحقق الشامل

```
Checklist النهائي قبل إبلاغ المدرسة:

التحقق التقني:
  □ تسجيل دخول المدير → يصل للـ Principal Portal
  □ تسجيل دخول معلم → يرى فصوله وطلابه فقط (لا غيرها)
  □ تسجيل جلسة تجريبية لطالب وهمي → تُحفَظ في DB
  □ عرض التشخيص → يعمل بعد جلستين < 60
  □ Logout → يخرج بنجاح

التحقق من الإعداد:
  □ عدد المعلمين صحيح: [X] معلم
  □ عدد الطلاب صحيح: [X] طالب
  □ عدد الفصول صحيح: [X] فصل
  □ كل معلم مرتبط بفصله الصحيح

بعد النجاح:
  □ أرسل Welcome Email الرسمي (Customer_Onboarding.md)
  □ جدول موعد تدريب المدير
  □ سجّل في CRM: Institution Setup Complete ✓
```

## الخطوة 7: إعداد الدعم المستمر

```
أضف المدرسة في:
  □ مجموعة واتساب الدعم (إذا وجدت)
  □ نظام Ticketing (اسم المدرسة + خطتها + تاريخ البدء)
  □ تقويم المتابعة الأسبوعية (30 يوماً أولى)
  □ تنبيه: تاريخ انتهاء الاشتراك (إشعار مسبق بـ 60 يوم)
```

## وقت الإعداد المستهدف

```
مدرسة < 100 طالب   : 2-3 ساعات
مدرسة 100-300 طالب : 3-4 ساعات
مدرسة 300-500 طالب : 4-6 ساعات
مجموعة مدارس       : يُقدَّر حسب الحجم

مع تحسين العملية (بعد 5 مدارس):
  الهدف: كل مدرسة ≤ 2 ساعة بمساعدة Automation
```
