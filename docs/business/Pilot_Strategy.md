## لماذا الـ Pilot مهم استراتيجياً؟

```
الـ Pilot يُحقق أربعة أهداف في آن واحد:

1. التحقق التقني   : هل النظام يعمل في بيئة حقيقية؟
2. التحقق من القيمة: هل المنتج يحل المشكلة التي نعتقدها؟
3. التحقق من السوق : هل المدارس مستعدة للدفع مقابل هذا؟
4. بناء Case Study  : قصة نجاح + أرقام حقيقية = أداة مبيعات

بدون Pilot ناجح: كل شيء مجرد افتراضات.
بعد Pilot ناجح: الاستراتيجية مبنية على بيانات.
```

## معايير اختيار مدرسة الـ Pilot

### خصائص المدرسة المثالية
```
الحجم:
  ✅ 100-300 طالب (راجع PILOT_PLAN.md)

الإدارة:
  ✅ مدير متحمس للتقنية — يؤمن بالبيانات
  ✅ لديه صلاحية القرار (لا يحتاج موافقات كثيرة)
  ✅ مستعد للمشاركة في تقرير نجاح (Case Study)

المعلمون:
  ✅ ≥ 3 معلمين مرنون ومنفتحون على تجربة جديدة
  ✅ ليسوا في أزمة (ضغط امتحانات، نزاعات إدارية)

التقنية:
  ✅ اتصال إنترنت مستقر في غرف الصف
  ✅ أجهزة (Tablets أو Laptops) متاحة أو يمكن توفيرها

التوقيت:
  ✅ بعيد عن موسم الامتحانات الكبرى (≥ شهرين قبلها)
  ✅ بعيد عن بداية العام (الفوضى الأولى)
```

### علامات تحذير — تجنبها
```
  ❌ مدير يقول "سنجرب ونرى" بدون التزام حقيقي
  ❌ مدرسة تتوقع "نتائج درامية في أسبوعين"
  ❌ ميزانية مضغوطة جداً (لا تستطيع حتى السعر الرمزي)
  ❌ معلمون عدائيون تجاه التكنولوجيا
  ❌ مشاكل إدارية داخلية حالية
```

## ديناميكية الـ Pilot

### قبل التوقيع: إدارة التوقعات
```
ما تقوله للمدرسة:
  ✅ "هذا Pilot — نتعلم معاً ونُحسّن معاً"
  ✅ "نتوقع أن يستغرق 8-12 أسبوعاً لرؤية نتائج واضحة"
  ✅ "سيحتاج المعلمون ساعتين للتدريب + 5 دقائق/جلسة بعد ذلك"
  ✅ "أنتم شركاء في بناء المنتج — ملاحظاتكم تُشكّل التطوير"

ما لا تقوله:
  ❌ "ستتحسن درجات الطلاب بالتأكيد في شهر"
  ❌ "المنصة ستُحل جميع مشاكل التعلم"
  ❌ "لا تحتاجون أي وقت إضافي" (كاذب)
```

### أثناء الـ Pilot: المشاركة النشطة
```
أسبوع 1-2: الاتصال كل يوم (مكثف)
  → حل أي مشكلة تقنية فوراً
  → تحقق: هل سجّلوا أول جلسة؟

أسبوع 3-8: اتصال أسبوعي (منتظم)
  → مكالمة 20 دقيقة مع مدير المدرسة
  → مراجعة الاستخدام: sessions created this week
  → اجمع ملاحظات المعلمين

أسبوع 9-12: جمع البيانات (تقييم)
  → Survey + مقابلات
  → حضب الأرقام من النظام
  → استعدّ لمحادثة التحويل

نقطة حرجة — أسبوع 4:
  إذا لم يسجّل ≥ 50% من المعلمين ولو جلسة واحدة:
  → اجتماع طارئ مع المدير فوراً
  → حدد المشكلة: تقنية؟ مقاومة؟ وقت؟
  → ابدأ إجراء تصحيحياً
  → إذا لم يتغير في أسبوع 6 → ناقش مع المدير إنهاء الـ Pilot بشكل محترم
```

### نهاية الـ Pilot: محادثة التحويل
```
الهدف: تحويل Pilot إلى اشتراك سنوي

التوقيت: أسبوع 10 (لا تنتظر أسبوع 12)

الأسلوب:
  1. قدّم تقرير النتائج (بياني وإيجابي)
     "معلموك سجّلوا 347 جلسة وكشفوا 28 فجوة — 18 منها حُلّت"
  2. اسأل: "كيف ترى هذه النتائج؟"
  3. اسأل: "ما الذي كنت تتمنى أن يكون مختلفاً؟"
  4. قدّم عرض الاشتراك: "نريدك أن تستمر بهذا السعر..."
  5. أعطِه أسبوعاً للقرار (لا تضغط)

إذا قال "نعم": ابدأ Institution_Onboarding.md الكامل
إذا قال "لاحقاً": حدّد موعد محادثة بعد شهر — لا تتركه بدون موعد
إذا قال "لا": اجمع السبب بصدق → راجع المنتج أو السعر أو التوقيت
```

## استراتيجية Pilot متعددة

```
بعد Pilot الأولى → كيف نُدير 5 Pilots في وقت واحد؟

المبدأ: لا تبدأ Pilot جديد قبل أن يكون الأول في أسبوع 6

جدول مقترح:
  Pilot 1: أسبوع 1-12 (مكثف)
  Pilot 2: يبدأ أسبوع 7 (بعد استقرار Pilot 1)
  Pilot 3: يبدأ أسبوع 13 (بعد إغلاق Pilot 1)
  ...

مع كل Pilot: نُحسّن عملية الـ Onboarding
  → Pilot 1: كل شيء يدوي
  → Pilot 3: نصف العملية موثَّق ويمكن تفويضه
  → Pilot 5: يمكن لشخص آخر في الفريق إدارته
```

## ما نتعلمه من كل Pilot

```
بعد كل Pilot — أجب على هذه الأسئلة قبل البداية التالية:

القرار الجوهري:
  □ هل المعلمون استخدموا المنصة بانتظام؟ لماذا أو لماذا لا؟
  □ ما الميزة التي طُلبت أكثر من مرة؟ → سجّلها في FUTURE_IDEAS.md
  □ ما الخطوة التي أضاعت وقت فريقنا أكثر؟ → أتمتها

التسعير:
  □ هل قبلوا السعر بسهولة؟ ربما السعر منخفض جداً
  □ هل ترددوا كثيراً؟ ربما السعر عالٍ أو القيمة غير واضحة

الـ Onboarding:
  □ ما الأسئلة التي تكررت؟ → أضفها للدليل
  □ ما الخطوة التي أخّرت البداية؟ → بسّطها

الـ Product:
  □ ما الذي أعجبهم أكثر؟ → أبرزه في المبيعات
  □ ما الذي لم يستخدموه؟ → لماذا؟
```
